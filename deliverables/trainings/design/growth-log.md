# Growth Log — Konzept & Umsetzung

## Was ist das Growth Log?

Das Growth Log ist das **persoenliche und Team-Logbuch fuer Wachstum** — ein lebendiges Dokument, in dem jede Person wöchentlich festhält, was sie ausprobiert hat, was dabei herauskam und wobei sie Unterstuetzung sucht. Im Team-Check-in wird daraus gemeinsam reflektiert und entschieden, was mit der ganzen Gruppe geteilt wird.

**Kernidee:** Das Growth Log ist kein Reporting-Tool und kein Leistungsnachweis. Es ist ein **Reflexionsinstrument** fuer persoenliches Wachstum — und ein **Verbindungsstueck** zwischen den Teammitgliedern, weil es sichtbar macht, wer wobei Hilfe braucht und wer etwas erlebt hat, das andere weiterbringt. Am Ende entsteht daraus die Growth Story des Jahrgangs.

**Keine Punkte, kein Scoring.** Der frueherer XP-Ansatz wurde bewusst gestrichen — er machte das System komplizierter als noetig und lenkte vom eigentlichen Zweck ab: echtes Wachstum sichtbar machen, nicht Punkte sammeln.

## Wo lebt das Growth Log?

**Ort: Mural-Board — dedizierter Bereich pro Team**

Jedes Team bekommt im Mural-Board einen eigenen, dauerhaft sichtbaren Growth-Log-Bereich. Dieser Bereich ist fuer alle Teams einsehbar (Transparenz), aber nur vom eigenen Team beschreibbar.

```
┌──────────────────────────────────────────────────────────────────┐
│                    GROWTH LOG ZONE                                │
├──────────────┬──────────────┬──────────────┬─────────────────────┤
│  Team Alpha  │  Team Beta   │  Team Gamma  │  Team Delta         │
│              │              │              │                     │
│  ┌────────┐  │  ┌────────┐  │  ┌────────┐  │  ┌────────┐        │
│  │ Woche 1│  │  │ Woche 1│  │  │ Woche 1│  │  │ Woche 1│        │
│  └────────┘  │  └────────┘  │  └────────┘  │  └────────┘        │
│  ┌────────┐  │  ┌────────┐  │  ┌────────┐  │  ┌────────┐        │
│  │ Woche 2│  │  │ Woche 2│  │  │ Woche 2│  │  │ Woche 2│        │
│  └────────┘  │  └────────┘  │  └────────┘  │  └────────┘        │
│  ...         │  ...         │  ...         │  ...                │
│              │              │              │                     │
└──────────────┴──────────────┴──────────────┴─────────────────────┘
```

**Warum Mural (und nicht ein Shared Doc)?**
- Visuell und raeumlich — Teams sehen auf einen Blick, wo sie stehen
- Sichtbar fuer alle — kein "verstecktes" Dokument in einem Team-Kanal
- Spielerisch gestaltbar — Farben, Icons, Milestone-Marker
- Niedrigschwellig — eine Sticky Note statt ein Absatz Text

## Aufbau einer Wochen-Karte

Jede Woche hat pro Team eine Karte mit vier persoenlichen Feldern und einem optionalen Team-Bereich:

```
┌────────────────────────────────────────────┐
│  WOCHE 3 — Team Alpha                     │
│  Keeper: Max                               │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │ 👤 Anna                              │  │
│  │ 🎯 "Copilot fuer echtes PR nutzen"   │  │
│  │ ✅ PR mit Copilot erstellt + Tests    │  │
│  │ 💡 "Test war perfekt — fuer die      │  │
│  │     falsche Funktion. Bester Fail."  │  │
│  │ 🤝 "Brauche Hilfe: Copilot in       │  │
│  │     Legacy-Codebase einsetzen"       │  │
│  └──────────────────────────────────────┘  │
│  ┌──────────────────────────────────────┐  │
│  │ 👤 Max                               │  │
│  │ 🎯 "Prompt-Library fuer Reviews"     │  │
│  │ ✅ 3 Review-Prompts geschrieben       │  │
│  │ 💡 "Chain-of-Thought macht Reviews   │  │
│  │     viel besser"                     │  │
│  │ 🤝 —                                 │  │
│  └──────────────────────────────────────┘  │
│  ... (Lisa, Tom)                           │
│                                            │
│  ── Optional: Team-Ziel ───────────────    │
│  "Alle 4 nutzen Copilot fuer ein PR"      │
│  → 3 von 4 geschafft                      │
└────────────────────────────────────────────┘
```

### Die vier persoenlichen Felder + optionales Team-Ziel

| Feld | Frage | Wann ausfuellen | Wer |
|------|-------|-----------------|-----|
| 🎯 **Mein Ziel** | *"Was will ich diese Woche zum ersten Mal mit AI ausprobieren?"* | Am Ende jedes Blocks (Conclusion) | Jede Person fuer sich |
| ✅ **Gemacht** | *"Was habe ich tatsaechlich getan?"* | Zwischen den Blocks, spaetestens vor dem Team-Check-in | Jede Person fuer sich |
| 💡 **Aha / Fail** | *"Was war mein bestes Aha oder lustigstes Fail?"* | Zwischen den Blocks oder im Team-Check-in | Jede Person fuer sich |
| 🤝 **Ich suche...** | *"Wobei brauche ich Unterstuetzung oder einen Sparringspartner?"* | Zwischen den Blocks oder im Team-Check-in | Jede Person fuer sich |
| 🏁 **Team-Ziel** *(optional)* | *"Gibt es etwas, das wir als Team gemeinsam ausprobieren wollen?"* | Am Ende jedes Blocks, wenn sich das Team einig ist | Team gemeinsam |

### Das 🤝-Feld: Warum es den Unterschied macht

Das "Ich suche..."-Feld ist das Herzstück des Team-Aspekts im Growth Log. Es macht das Team wertvoll — auch ohne gemeinsames Ziel:

- **Sichtbarkeit fuer Bedarfe:** *"Ich komme bei X nicht weiter"* oder *"Hat jemand Erfahrung mit Y?"* — das Team sieht, wo es fuereinander da sein kann
- **Niedrigschwellige Hilfe-Anfrage:** Einfacher, eine Sticky Note zu schreiben als im Meeting zu fragen
- **Natuerliche Zusammenarbeit:** Wer sieht, dass jemand Hilfe braucht, kann sich melden — ohne Zuweisung, ohne Pflicht
- **Basis fuer Buddies und Pairing:** Aus dem 🤝-Feld ergeben sich natuerlich Lern-Buddy-Konstellationen

**Beispiele:**
- *"Ich brauche jemanden zum gemeinsamen Ausprobieren von Copilot im Pair Programming"*
- *"Hat jemand einen guten Prompt fuer Meeting-Protokolle? Meine funktionieren nicht"*
- *"Wuerde gerne mit jemandem zusammen einen CRAFT Explorer Walkthrough machen"*
- *"Suche Sparringspartner fuer mein erstes AI-Transformationsgespraech beim Kunden"*
- Oder einfach: *"—"* (kein Bedarf diese Woche — auch okay)

## Der Team-Check-in: Das Kern-Ritual (15 Min vor jedem Block)

### Idee

Statt Show & Tell im Plenum kalt zu starten, treffen sich die Teams **15 Minuten vor dem offiziellen Block-Beginn** im Team-Check-in. Dort besprechen sie ihr Growth Log — und entscheiden gemeinsam, was sie im Show & Tell mit der ganzen Gruppe teilen wollen.

### Warum 15 Minuten vorher?

- **Keine Block-Zeit verbraucht** — die 60 Min des Blocks bleiben vollstaendig fuer 4-Cs
- **Selbstorganisiert** — das Team trifft sich eigenstaendig, der Trainer ist nicht dabei
- **Niedrigschwellig** — im kleinen Kreis faellt Teilen und Reflektieren leichter als im Plenum
- **Kuratierte Show & Tell** — statt spontanem "Wer moechte was sagen?" kommen vorbereitete, gute Beitraege

### Ablauf des Team-Check-ins (15 Min)

```
┌────────────────────────────────────────────────────┐
│  TEAM CHECK-IN (15 Min vor Block-Beginn)           │
│                                                    │
│  📋 Runde 1: "Was habe ich gemacht?" (5 Min)       │
│     Jeder teilt kurz: ✅ Gemacht + 💡 Aha/Fail     │
│     Reihum, je ~1 Min pro Person                   │
│                                                    │
│  🤝 Runde 2: "Wer braucht was?" (5 Min)            │
│     Wer hat ein 🤝 eingetragen?                    │
│     Kann jemand im Team helfen?                    │
│     → Kurze Verabredungen treffen                  │
│                                                    │
│  🎤 Runde 3: "Was teilen wir?" (5 Min)             │
│     Team waehlt: Welches Aha oder Fail ist so      │
│     gut, dass es die ganze Gruppe hoeren sollte?   │
│     → 1 Person bereitet 2 Min Show & Tell vor      │
│                                                    │
└────────────────────────────────────────────────────┘
```

**Runde 1 — "Was habe ich gemacht?"** (5 Min)
- Reihum: Jeder teilt kurz Gemacht + Aha/Fail (je ~1 Min)
- Growth Log als Spickzettel — wer vorher eingetragen hat, muss nicht improvisieren
- Keeper moderiert (oder wer gerade will)

**Runde 2 — "Wer braucht was?"** (5 Min)
- 🤝-Feld durchgehen: Wer hat etwas eingetragen?
- Kurzer Austausch: *"Das kenne ich — lass uns am Donnerstag 15 Min zusammensetzen"*
- Oder: *"Das weiss ich auch nicht — aber vielleicht koennen wir das Team im Plenum fragen?"*
- Hier entstehen natuerliche Buddy-Momente und Pairing-Verabredungen

**Runde 3 — "Was teilen wir?"** (5 Min)
- Team waehlt den besten Beitrag fuer Show & Tell (2 Min im Plenum)
- Das kann ein Aha, ein Fail, ein Hilfe-Gesuch an die Gruppe oder ein Erfolg sein
- Eine Person uebernimmt: *"Ich erzaehl das gleich fuer uns"*

### Was der Check-in bewirkt

| Ohne Check-in | Mit Check-in |
|---------------|-------------|
| Show & Tell: Stille, keiner traut sich | Show & Tell: vorbereitete, kuratierte Beitraege |
| Growth Log: "Das mach ich spaeter" | Growth Log: "Muss vor dem Check-in drin sein" |
| Team: Einzelkaempfer mit gleichem Logo | Team: weiss voneinander, hilft einander |
| Hilfe-Bedarf: unsichtbar | Hilfe-Bedarf: sichtbar und adressierbar |

## Wie wird es selbstorganisiert?

### Das Problem: Pflege-Aufwand ohne externen Druck

Growth Logs sterben, wenn sie sich wie Pflicht anfuehlen. Die Loesung: **minimaler Aufwand, maximale Sichtbarkeit, ein festes Ritual das den Rhythmus traegt.**

### Die Loesung: Drei Mechanismen

#### 1. Growth-Log-Keeper (pro Team, rotierend)

Jedes Team bestimmt pro Woche einen **Growth-Log-Keeper**. Rotation, kein Daueramt.

**Aufgabe des Keepers:**
- Erinnert das Team zwischen den Sessions kurz: *"Tragt bitte eure Sachen ins Growth Log ein — Check-in ist am [Tag] um [Uhrzeit]."*
- Moderiert den Team-Check-in (oder delegiert)
- Stellt sicher, dass die Wochen-Karte vor dem Check-in ausgefuellt ist
- Hat NICHT die Aufgabe, alles selbst zu schreiben

**Warum rotierend?**
- Kein Burnout auf einer Person
- Jeder lernt die Keeper-Perspektive kennen
- Wer Keeper war, versteht den Wert des Logs besser

**Einfuehrung:** Block 1 (Conclusion) — Trainer sagt: *"Jedes Team waehlt jetzt einen Growth-Log-Keeper fuer diese Woche. Naechste Woche rotiert. Der Keeper erinnert euch einmal und moderiert den Check-in — mehr nicht."*

#### 2. Der Team-Check-in als tragendes Ritual

Das Growth Log wird nicht "zusaetzlich" gepflegt, sondern der **15-Minuten-Check-in vor jedem Block** gibt den Rhythmus vor:

| Moment | Was passiert mit dem Growth Log | Dauer |
|--------|--------------------------------|-------|
| **Conclusion (Ende jedes Blocks)** | Jeder setzt persoenliches Ziel → 🎯 Mein Ziel | 2 Min |
| **Zwischen den Blocks** | Jeder traegt ein: ✅ Gemacht, 💡 Aha/Fail, 🤝 Ich suche... | ~5 Min ueber die Woche verteilt |
| **Team-Check-in (15 Min vor Block)** | Team bespricht Growth Log → waehlt Show-&-Tell-Beitrag | 15 Min |
| **Show & Tell (Beginn des Blocks)** | 1 kuratierter Beitrag pro Team — vorbereitet im Check-in | 5 Min |
| **Block 5 (Halbzeit)** | Kurzer Rueckblick: Wo standen wir, wo stehen wir? | 3 Min |
| **Block 10 (Community Day)** | Growth Story Ernte — die besten Eintraege werden zur Jahrgangs-Story | 10 Min |

**Schluessel-Insight:** Der Check-in ist der Motor. Wer sein Growth Log nicht pflegt, hat beim Check-in nichts zu besprechen — das ist die natuerliche Konsequenz, kein Zwang. Und weil das Team sich trifft, entsteht sozialer Kontext: man will etwas beizutragen haben.

#### 3. Sichtbarkeit erzeugt Motivation

- Alle Growth Logs sind im Mural fuer alle Teams sichtbar
- Der Trainer hebt zu Beginn jedes Blocks den **staerksten Wachstumsmoment der Vorwoche** hervor (1 Min, Trainer-Nominierung oder Peer-Tipp)
- Kein Ranking, aber: *"Schaut mal, was Team Beta diese Woche zum ersten Mal gemacht hat — das inspiriert"*

## Spielerischer Kontext: Wachstum statt Wettbewerb

### Design-Prinzipien

| Prinzip | Umsetzung | Vermeidung |
|---------|-----------|------------|
| **Wachstum feiern** | "Zum ersten Mal" als Kern-Metrik | Keine Punkte, kein Scoring |
| **Persoenliche Ziele** | Jeder setzt sein eigenes Ziel pro Woche | Keine externen Vorgaben oder Benchmarks |
| **Fails = Wachstum** | Geteilte Fails sind ausdruecklich erwuenscht | Kein Blame, kein "warum hat das nicht geklappt" |
| **Verbindung statt Vergleich** | 🤝-Feld macht Zusammenarbeit natuerlich | Kein Ranking, kein Leaderboard |
| **Transparenz ohne Druck** | Alle sehen alle Logs, aber es gibt keine Bewertung | Kein Podium, keine Gewinner-Verkuendung |

### Spielerische Elemente in Mural

**Wachstum sichtbar machen — ohne Punkte:**
Statt XP-Balken zeigen wir Wachstum visuell durch die **Dichte** der Growth-Log-Karten und Milestone-Marker. Ein volles, buntes Growth Log erzaehlt eine Geschichte — das ist befriedigender als eine Zahl.

**Reframing durch Sprache:**
- Nicht "Leistung" sondern "Wachstum"
- Nicht "Bewertung" sondern "Reflexion"
- Nicht "Wettbewerb" sondern "Entdeckungsreise"
- Nicht "Pflicht" sondern "Ritual"

**Wachstumsmomente statt Rankings:**
Der Trainer hebt nicht "das beste Team" hervor, sondern den **mutigsten Versuch**, das **ehrlichste Fail** oder den **ueberraschendsten Aha-Moment** — unabhaengig vom Team.

### Milestone-Marker (optional)

Teams koennen auf ihrer Growth-Log-Spur Milestones platzieren — kleine visuelle Marker fuer besondere Momente:

| Marker | Bedeutung |
|--------|-----------|
| 🌱 | Erstes Mal: etwas Neues zum ersten Mal ausprobiert |
| 💥 | Epic Fail: ein besonders lehrreicher Fehlschlag |
| 🔗 | Buddy-Moment: zusammen mit jemandem entstanden |
| 🎤 | Practice Story: eigene Erfahrung mit der Gruppe geteilt |
| 🏔️ | Gipfelmoment: persoenlicher Durchbruch |

Diese Marker machen das Log visuell ansprechend und erzaehlen eine Geschichte — ohne Zahlen oder Rankings.

## Integration in die Bloecke

### Block 1 (Kickoff): Growth Log einfuehren

**Wo:** Concept-Phase (bei "Growth Story statt Leistungsdruck") + Conclusion

**Was der Trainer erklaert:**
> "Jedes Team fuehrt ab heute ein Growth Log — euer Team-Logbuch im Mural. Pro Woche traegt jeder von euch vier Dinge ein: Was habe ich mir vorgenommen? Was habe ich gemacht? Was war mein Aha oder Fail? Und — wobei suche ich Unterstuetzung vom Team?"
>
> "Vor jedem Block trefft ihr euch 15 Minuten vorher im Team-Check-in. Dort besprecht ihr eure Eintraege und entscheidet, was ihr im Show & Tell mit allen teilen wollt. Das ist euer Ritual — selbstorganisiert, ohne Trainer."
>
> "Und: Wer moechte, kann ein gemeinsames Team-Ziel setzen — aber das ist optional. Das persoenliche Ziel ist Pflicht, das Team-Ziel eine Kuer fuer Teams, die sich einig sind."
>
> "Waehlt jetzt einen Growth-Log-Keeper fuer diese Woche. Der Keeper erinnert euch einmal und moderiert den Check-in. Naechste Woche rotiert."

**Was passiert:**
- Teams sehen ihre Growth-Log-Zone im Mural
- Teams waehlen ersten Keeper
- Jeder setzt ein erstes persoenliches 🎯 Mein Ziel: *"Was will ich naechste Woche zum ersten Mal mit AI ausprobieren?"*
- Optional: Team-Ziel, falls sich das Team einig ist

### Block 2–9: Woechentlicher Rhythmus

1. **Team-Check-in (15 Min vor Block):** Team bespricht Growth Log — Gemacht, Aha/Fail, Unterstuetzungs-Bedarfe → waehlt Show-&-Tell-Beitrag
2. **Show & Tell (5 Min Beginn):** 1 kuratierter Beitrag pro Team (im Check-in vorbereitet)
3. **Trainer Highlight (1 Min):** Staerkster Wachstumsmoment der Vorwoche
4. **Conclusion (2 Min):** Jeder setzt neues persoenliches Ziel → 🎯 Mein Ziel + neuer Keeper bestimmen. Optional: Team-Ziel.

### Block 5 (Halbzeit): Growth Log Review

- Kurzer gemeinsamer Rueckblick: *"Was hat sich veraendert seit Block 1?"*
- Teams reflektieren: *"Was haben wir uns vorgenommen vs. was haben wir wirklich getan?"*
- 🤝-Rueckblick: *"Welche Zusammenarbeit ist aus den 'Ich suche...'-Eintraegen entstanden?"*
- Optional: Teams duerfen ihre Ziel-Strategie anpassen

### Block 10 (Community Day): Growth Story Ernte

- Teams praeparieren aus ihrem Growth Log: *"Unsere 3 staerksten Wachstumsmomente"*
- Daraus entsteht die **Growth Story des Jahrgangs** — eine Collage aus Ahas, Fails, Erstmals-Momenten und Durchbruechen
- Diese Story kann in die Community-Bibliothek: Inspiration fuer die naechste Kohorte

## FAQ

**"Was wenn ein Team das Growth Log nicht pflegt?"**
Kein Zwang. Aber: Der Check-in bezieht sich aufs Growth Log. Wer nichts eingetragen hat, hat weniger zu besprechen. Der natuerliche soziale Kontext reicht meist — kein Tadel noetig. Falls es dauerhaft passiert: Trainer spricht das Team unter vier Augen an.

**"Wie viel Zeit kostet das Growth Log pro Woche?"**
~5 Minuten pro Person (Sticky Notes eintragen) + 15 Minuten Team-Check-in. Keine langen Texte — Stichworte reichen.

**"Muessen alle den Check-in mitmachen?"**
Ja — der Check-in ist das tragende Ritual. Wer nicht kann, traegt vorher ins Growth Log ein und informiert den Keeper. Das Team kann den Beitrag trotzdem einbeziehen.

**"Was wenn niemand etwas im 🤝-Feld eintraegt?"**
Voellig okay. Es gibt Wochen, in denen man keine Hilfe braucht. Der Wert entsteht schon dadurch, dass die Moeglichkeit da ist und regelmaessig danach gefragt wird. Sobald es einmal genutzt wird und eine gute Zusammenarbeit entsteht, traegt sich das Feld von selbst.

**"Brauchen wir wirklich einen Keeper?"**
Der Keeper ist kein Manager, sondern ein Erinnerungs-Anker. Ohne eine benannte Person passiert erfahrungsgemäss nichts — weil alle denken, jemand anders macht es. Der Keeper loest dieses Diffusion-of-Responsibility-Problem. Rotation verhindert, dass es zur Last wird.

**"Koennen wir statt Mural ein Shared Doc nutzen?"**
Moeglich, aber Mural hat Vorteile: visueller, niedrigschwelliger (Sticky Note vs. Absatz), transparent fuer alle auf einen Blick. Ein Shared Doc neigt dazu, "unsichtbar" zu werden. Falls Mural-Zugang ein Problem ist: Google Doc mit klarer Struktur als Fallback.

**"Team-Ziel: Pflicht oder optional?"**
Optional. Der Fokus liegt auf persoenlichen Zielen — die sind verbindlich (jeder setzt eins). Ein Team-Ziel ist Kuer: wenn sich das Team auf etwas Gemeinsames einigen kann, umso besser. Aber kein Zwang. Manche Wochen passt es, manche nicht.
